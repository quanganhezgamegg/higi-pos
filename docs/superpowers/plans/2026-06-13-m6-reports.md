> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

Rust tests: seed in-memory DB bằng `migrations::run` + DML trực tiếp, assert aggregate numbers.

**Tech Stack:** rusqlite + rusqlite_migration (sẵn), React + shadcn/ui (`card`, `table`, `select`, `input`), cargo test (TDD aggregate queries), không thêm dep mới.

- [ ] **Step 1: Tạo `src-tauri/migrations/0007_report_views.sql`**

```sql
-- VIEW tổng hợp đơn đã thanh toán (PAID) với cột ngày thanh toán dạng TEXT YYYY-MM-DD
-- Dùng cho báo cáo theo ngày; paid_date lấy 10 ký tự đầu của paid_at (ISO-8601 UTC).
CREATE VIEW IF NOT EXISTS v_paid_orders AS
SELECT
  o.id,
  o.code,
  o.shift_id,
  o.subtotal,
  o.discount_total,
  o.total,
  o.paid_at,
  substr(o.paid_at, 1, 10) AS paid_date
FROM orders o
WHERE o.status = 'PAID';

-- VIEW tóm tắt thanh toán theo đơn và phương thức
CREATE VIEW IF NOT EXISTS v_payment_summary AS
SELECT
  p.order_id,
  p.method,
  SUM(p.amount) AS method_total,
  COUNT(*)      AS method_count
FROM payments p
JOIN orders o ON o.id = p.order_id
WHERE o.status = 'PAID'
GROUP BY p.order_id, p.method;

-- VIEW tổng hợp dòng món (chỉ đơn PAID) cho top-products
CREATE VIEW IF NOT EXISTS v_paid_order_items AS
SELECT
  oi.id,
  oi.order_id,
  oi.product_name,
  oi.quantity,
  oi.line_total,
  o.shift_id,
  substr(o.paid_at, 1, 10) AS paid_date
FROM order_items oi
JOIN orders o ON o.id = oi.order_id
WHERE o.status = 'PAID';
```

```rust
use serde::{Deserialize, Serialize};

/// Input cho tất cả report commands.
/// Chấp nhận date range (from/to) HOẶC shift_id — dùng xen kẽ loại trừ nhau.
/// - Date range: from/to là TEXT YYYY-MM-DD (inclusive). Nếu chỉ truyền from → from..=today.
/// - Shift mode: chỉ shift_id, bỏ from/to.
#[derive(Debug, Clone, Deserialize)]
pub struct ReportRangeInput {
    pub from: Option<String>,  // YYYY-MM-DD
    pub to: Option<String>,    // YYYY-MM-DD
    pub shift_id: Option<i64>,
}

/// Kết quả tổng hợp — dùng cho cả date-range và per-shift.
#[derive(Debug, Clone, Serialize)]
pub struct SalesSummary {
    pub revenue: i64,           // Σ orders.total WHERE status='PAID'
    pub order_count: i64,       // COUNT(*)
    pub avg_order_value: i64,   // revenue / order_count (0 khi order_count=0)
    pub discount_total: i64,    // Σ orders.discount_total WHERE status='PAID'
}

/// Một hàng trong bảng cơ cấu thanh toán.
#[derive(Debug, Clone, Serialize)]
pub struct PaymentMixRow {
    pub method: String,  // "CASH" hoặc "QR"
    pub total: i64,      // Σ payments.amount cho method này
    pub count: i64,      // số lần thanh toán
}

/// Một hàng trong bảng top sản phẩm.
#[derive(Debug, Clone, Serialize)]
pub struct TopProductRow {
    pub product_name: String,
    pub quantity: i64,   // Σ order_items.quantity
    pub revenue: i64,    // Σ order_items.line_total
}
```

> **Lưu ý `PaymentMixRow.method`:** contract §3 khai báo `pub method: PaymentMethod` (enum); tuy nhiên vì `reports.rs` không có dependency vào `domain/payments.rs`, dùng `String` để tránh import chéo và giữ module độc lập. FE nhận `"CASH"` hoặc `"QR"` — không thay đổi về wire format.

Đây là phần trọng tâm. Mỗi query được viết test trước (seed data thủ công bằng DML → gọi hàm → assert số aggregate) rồi mới viết impl. Tất cả hàm nhận `&Connection`, trả `rusqlite::Result<T>`, theo mẫu `repo/menu.rs`.

### SQL aggregate queries (nguồn sự thật)

**Q1 — `report_sales_summary` (date range):**
```sql
SELECT
  COALESCE(SUM(total), 0)          AS revenue,
  COUNT(*)                          AS order_count,
  COALESCE(SUM(discount_total), 0) AS discount_total
FROM orders
WHERE status = 'PAID'
  AND substr(paid_at, 1, 10) >= ?1   -- from (YYYY-MM-DD)
  AND substr(paid_at, 1, 10) <= ?2   -- to   (YYYY-MM-DD)
```
`avg_order_value = if order_count > 0 { revenue / order_count } else { 0 }` — tính trong Rust.

**Q2 — `report_sales_summary` (shift mode):**
```sql
SELECT
  COALESCE(SUM(total), 0)          AS revenue,
  COUNT(*)                          AS order_count,
  COALESCE(SUM(discount_total), 0) AS discount_total
FROM orders
WHERE status = 'PAID'
  AND shift_id = ?1
```

**Q3 — `report_payment_mix` (date range):**
```sql
SELECT
  p.method,
  COALESCE(SUM(p.amount), 0) AS total,
  COUNT(*)                    AS count
FROM payments p
JOIN orders o ON o.id = p.order_id
WHERE o.status = 'PAID'
  AND substr(o.paid_at, 1, 10) >= ?1
  AND substr(o.paid_at, 1, 10) <= ?2
GROUP BY p.method
ORDER BY p.method
```

**Q4 — `report_payment_mix` (shift mode):**
```sql
SELECT
  p.method,
  COALESCE(SUM(p.amount), 0) AS total,
  COUNT(*)                    AS count
FROM payments p
JOIN orders o ON o.id = p.order_id
WHERE o.status = 'PAID'
  AND o.shift_id = ?1
GROUP BY p.method
ORDER BY p.method
```

**Q5 — `report_top_products` (date range):**
```sql
SELECT
  oi.product_name,
  SUM(oi.quantity)   AS quantity,
  SUM(oi.line_total) AS revenue
FROM order_items oi
JOIN orders o ON o.id = oi.order_id
WHERE o.status = 'PAID'
  AND substr(o.paid_at, 1, 10) >= ?1
  AND substr(o.paid_at, 1, 10) <= ?2
GROUP BY oi.product_name
ORDER BY SUM(oi.quantity) DESC, SUM(oi.line_total) DESC
LIMIT ?3
```

**Q6 — `report_top_products` (shift mode):**
```sql
SELECT
  oi.product_name,
  SUM(oi.quantity)   AS quantity,
  SUM(oi.line_total) AS revenue
FROM order_items oi
JOIN orders o ON o.id = oi.order_id
WHERE o.status = 'PAID'
  AND o.shift_id = ?1
GROUP BY oi.product_name
ORDER BY SUM(oi.quantity) DESC, SUM(oi.line_total) DESC
LIMIT ?2
```

**Q7 — `report_discount_total` (date range):**
```sql
SELECT COALESCE(SUM(discount_total), 0)
FROM orders
WHERE status = 'PAID'
  AND substr(paid_at, 1, 10) >= ?1
  AND substr(paid_at, 1, 10) <= ?2
```

**Q8 — `report_discount_total` (shift mode):**
```sql
SELECT COALESCE(SUM(discount_total), 0)
FROM orders
WHERE status = 'PAID'
  AND shift_id = ?1
```

```tsx
        reportSalesSummary(payload),
        reportPaymentMix(payload),
        reportTopProducts(payload, 10),
      ])
      setSummary(s)
      setPaymentMix(pm)
      setTopProducts(tp)
    } catch (e) {
      setError(String(e))
    } finally {
      setLoading(false)
    }
  }, [from, to])

  return (
    <div className="mx-auto flex max-w-4xl flex-col gap-6 p-6">
      <div className="flex items-center justify-between">
        <h1 className="text-2xl font-bold">Báo cáo Doanh thu</h1>
        <Button variant="outline" asChild>
          <Link to="/">Về trang chủ</Link>
        </Button>
      </div>

      {/* Bộ lọc ngày */}
      <div className="flex flex-wrap items-end gap-4 rounded-md border p-4">
        <div className="flex flex-col gap-1">
          <Label htmlFor="from">Từ ngày</Label>
          <Input
            id="from"
            type="date"
            value={from}
            onChange={(e) => setFrom(e.target.value)}
            className="w-40"
          />
        </div>
        <div className="flex flex-col gap-1">
          <Label htmlFor="to">Đến ngày</Label>
          <Input
            id="to"
            type="date"
            value={to}
            onChange={(e) => setTo(e.target.value)}
            className="w-40"
          />
        </div>
        <Button onClick={run} disabled={loading} className="min-h-[44px] min-w-[100px]">
          {loading ? "Đang tải…" : "Xem báo cáo"}
        </Button>
      </div>

      {error && <p className="rounded-md bg-destructive/10 p-3 text-sm text-destructive">{error}</p>}

      {summary && (
        <>
          <SummaryCards summary={summary} />

          <section>
            <h2 className="mb-3 text-lg font-semibold">Top sản phẩm bán chạy</h2>
            <TopProductsTable rows={topProducts} />
          </section>

          <section>
            <h2 className="mb-3 text-lg font-semibold">Cơ cấu thanh toán</h2>
            <PaymentMixTable rows={paymentMix} />
          </section>
        </>
      )}
    </div>
  )
}
```
